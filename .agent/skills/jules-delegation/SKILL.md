# Jules Delegation Skill

AntigravityからJulesへタスクを委任するためのスキル。レート制限対策と作業分担を実現する。

## 使い方

### タスクを委任する

ユーザーからの作業依頼を受けたとき、以下の条件でJulesへの委任を検討：

1. **レート制限に近づいている場合**
2. **独立した実装タスク**（例：テスト作成、ドキュメント更新、リファクタリング）
3. **時間のかかるタスク**で並列実行が効果的な場合

### 委任コマンド

```bash
# タスクを委任
python -m src.task_dispatcher delegate "タスクの説明"

# ステータス確認
python -m src.task_dispatcher status
```

### ワークフロー `/delegate` の使用

ワークフローファイル `.agent/workflows/delegate.md` を参照してタスクを委任。

---

## 委任判断基準

| 委任すべきタスク               | 委任すべきでないタスク     |
| ------------------------------ | -------------------------- |
| テスト作成                     | 設計判断が必要な作業       |
| ドキュメント更新               | ユーザーとの対話が必要     |
| 定型的なリファクタリング       | コンテキスト依存が高い作業 |
| 新規ファイル作成（仕様が明確） | レビューや承認が必要な変更 |

---

## 結果の確認

Julesからの結果を受け取ったら：

1. **コード確認**: 生成されたコードをレビュー
2. **テスト実行**: 必要に応じてテストを実行
3. **マージ判断**: 問題なければプロジェクトに統合

---

## 設定

プロジェクトルートに `config.json` が必要：

```json
{
  "jules_api_key": "YOUR_API_KEY"
}
```

※ リポジトリとブランチは自動検出されます

## スクリプト

`scripts/` ディレクトリに補助スクリプトを配置：

- `delegate.py` - タスク委任の実行
- `status.py` - ステータス確認
